# AI-TDD Test Plan

사람이 다음 동작과 기대 결과를 승인하면 AI 코딩 에이전트가 미완료 항목 하나만 선택한다. 완료 표시는 유효한 RED, focused GREEN, full suite GREEN을 모두 확인한 뒤에만 바꾼다.

## Completed

- [x] 정상 예약 생성은 `201`과 생성된 예약을 반환한다.
- [x] 종료 시간이 시작 시간보다 늦지 않으면 `422`를 반환한다.
- [x] 같은 방의 겹치는 예약은 `409`를 반환한다.
- [x] 빈 `room_id`는 `422`로 거절한다.
- [x] 생성된 예약 목록을 조회한다.
- [x] 같은 방의 인접 예약은 허용한다.
- [x] 시간이 겹쳐도 방이 다르면 허용한다.
- [x] FastAPI root에서 발표 화면을 제공한다.
- [x] 발표 화면에서 1차 자료와 OpenAPI 문서를 연결한다.
- [x] 발표 화면에서 AI-TDD 해설 영상과 프로젝트 흐름을 연결한다.

인접 예약과 다른 방 테스트는 새 구현을 끌어낸 RED가 아니라 일반화된 충돌 조건을 고정한 계약 테스트다. 각 사이클의 실제 출력은 `docs/TDD_CYCLES.md`에 기록한다.

## Next Item Protocol

1. 사람이 다음 동작과 기대 결과를 승인한다.
2. AI가 미완료 항목 하나를 진행 중으로 표시한다.
3. 가장 작은 실행 가능한 테스트 하나를 작성한다.
4. 요구 동작 부재로 실패하는 유효한 RED를 확인한다.
5. 테스트를 수정하지 않고 최소 제품 코드로 GREEN을 만든다.
6. focused 테스트와 full suite를 순서대로 실행한다.
7. GREEN 상태에서만 구조 변경을 별도로 수행하고 full suite를 다시 실행한다.
8. 사람이 diff와 설계를 검토한 뒤 항목을 완료한다.
