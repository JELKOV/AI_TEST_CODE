AI-assisted TDD (AI 보조 테스트 주도 개발)

AI가 구현 속도를 높이고, 테스트가 행동을 통제한다

OpenAPI Docs (API 문서) uv run pytest -q
AI

AI Agent (AI 에이전트)

테스트 목록을 따라 한 번에 한 행동만 구현하고 실행 증거를 남긴다.

H

Human (사람)

예약 정책과 기대 결과를 결정하고 테스트와 변경 범위를 검토한다.

T

pytest (테스트 실행기)

합의된 행동을 실행 가능한 규칙으로 고정하고 회귀를 즉시 드러낸다.

TDD Cycle (테스트 주도 흐름)

RED → GREEN → REFACTOR

Canon TDD (원문) FastAPI Testing (공식 문서)
01

RED (실패 확인)

겹치는 예약은 409여야 한다는 테스트를 먼저 쓰고, 정책 부재로 실패하는지 확인한다.

assert overlapping.status_code == 409
02

GREEN (최소 구현)

같은 방에서 두 시간 구간이 교차할 때만 거절하는 최소 조건을 구현한다.

new.start < saved.end and saved.start < new.end
03

REFACTOR (구조 개선)

모든 테스트가 통과한 상태에서 동작은 유지하고 이름과 구조만 개선한다.

uv run pytest -q
